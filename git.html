Git Nedir ve Temel Komutları

1. Git Repository (Repo) Nedir?

Git repository (repo), bir projenin tüm dosyalarını, geçmiş sürümlerini ve bu dosyalarda yapılan değişikliklerin kaydını tutan bir veri deposudur. Git ile bir repo oluşturduğunuzda, her dosya ve değişiklik tarihçesiyle birlikte saklanır.

  • git init: Yeni bir Git deposu başlatmak.
  • git clone [repo-url]: Uzak bir Git reposunun bir kopyasını almak.

2. Git Commit

{/* Paragraf içindeki komut adı düz metin olarak değiştirildi */}

"git commit" komutu, yapılan değişiklikleri kaydetmek için kullanılır. Git, değişiklikleri takip eder ve bunları kaydetmek için commit (yazılımın anlık durumu) oluşturur.

  • git add [dosya-adi]: Değiştirilen dosyaları staging alanına ekler.
  • git commit -m "Mesaj": Staging alanındaki dosyalar için bir commit mesajı ile değişiklikleri kaydeder.
  • git status: Çalışma alanındaki ve staging alanındaki dosyaların durumunu gösterir.

3. Git Push

"git push" komutu, yerel (local) değişikliklerinizi uzak (remote) depoya göndermek için kullanılır. Bu işlem, başkalarının da bu değişikliklere erişmesini sağlar.

  • git push [remote] [branch]: Uzak depoya değişikliklerinizi gönderir.

4. Git Pull

"git pull" komutu, uzak depodaki değişiklikleri yerel bilgisayarınıza alır. Bu komut, başkalarının yaptığı değişiklikleri almak için kullanılır ve yerel deponuzu güncel tutar.

  • git pull [remote] [branch]: Uzak depodan yerel deponuza değişiklikleri alır.

5. Git Branch

"git branch" komutu, farklı iş akışları veya özellikler için farklı dallar (branches) oluşturmanıza ve yönetmenize olanak tanır.

  • git branch: Mevcut dalları listelemek.
  • git checkout [branch-adi]: Farklı bir dala geçmek.
  • git merge [branch-adi]: Bir dalı başka bir dala birleştirir.

6. Git Log

"git log" komutu, commit geçmişini görmek için kullanılır. Bu komut, proje üzerindeki tüm commit'leri ve bunların detaylarını gösterir.

  • git log: Commit geçmişini gösterir.

7. Git Reset

"git reset" komutu, önceki bir commit'e dönmek için kullanılır. Bu komut, yapılan değişiklikleri geri almak için yararlıdır.

  • git reset --hard [commit-id]: Verilen commit'e geri dönmenizi sağlar.

8. Git Diff

"git diff" komutu, yapılan değişikliklerin karşılaştırılmasını sağlar. Bu komut, hangi satırların değiştiğini ve ne tür değişiklikler yapıldığını gösterir.

  • git diff: Çalışma alanındaki değişiklikleri gösterir.
